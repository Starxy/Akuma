使用 J1800 工控板搭建家用服务器

之前发现家里联通宽带有公网 ip 感觉有点搞头就买了个 j1800 的工控板子扔家里做了服务器，每天耗电大概在 10W 左右，主要负载了一个 nextcloud 做家用网盘，一个 TeamSpeak3 服务器用来玩游戏开语音，一个用于局域网联机的 VPN 网络， 目前运行稳定，基本没有出现过服务器宕机的情况。以下用来记录部署使用过程中遇到的一些问题和解决办法。有时间的话可以介绍各个服务的搭建过程。

部署服务

  • 网盘：Nextcloud
  • 媒体系统：Emby
  • 图床：Elimage
  • 团队交流：TeamSpeak3、Mattermost
  • 数据库系统：mariadb、redis、mongodb
  • 游戏服务器：dst、csgo、minecraft、terraria
  • VPN：Tinc

使用 Certbot 对动态解析域名开启 HTTPS

动态域名的解决方案就很随意了，现在有很多成熟的动态 ip 解析办法，我手里自有一个域名，所以只需要检测 ip 地址变更的时候向 dnspod 提交变更解析的申请即可。详细文档可以参考 dnspod api。

对于家用的宽带，普遍的情况是运营商禁用了常用的低位端口，22、80、443 这些端口不用考虑根本不可能开放的。而 Certbot 默认情况验证方式为：

  • certbot 程序在 web 目录的根目录下放置一个文件。
  • letsencrypt 的服务器通过域名来访问这个文件，来验证你申请的域名是属于你的

然而默认是通过 80/443 端口来访问，所以在家庭带宽情况下该验证方法就失效了。我们可以通过修改域名 DNS 的 TXT 记录来验证域名

运行 sudo certbot --manual --preferred-challenges dns certonly 会要求你输入需要启用 HTTPS 的域名然后同意记录本机 ip 后弹出以下提示：

根据上述提示，修改为于域名添加一个 TXT 记录，主机记录为 _acme-challenge，记录值为上述提示的字符串 z5NOLEpGmoL7MZiOYVaD0H-195uIVZstnHmEsI1ZakI。等到域名生效之后在命令行中按回车继续。等待 letsencrypt 验证成功即可。

该中方法不支持 cerbot renew，请等到域名快过期后（三个月）手动再次执行上述操作，或者手动编写脚本来对 dns 值进行自动修改。

Netdata 错误处理

10min netdev budget ran outs

这是一个与数据包接收与处理有关的问题。根据 softnet 图表描述，该图表有三个值

  • processed 代表 cpu 处理的数据包
  • dropped 代表丢弃的包计数，该出丢包是由于内核缓存队列满了，可以通过增加 net.core.netdev_max_backlog 的值修复
  • squeezed 代表在从 ring buffer 运输到内核网络缓存队列时, 由于“运输小车”没能装下,而导致丢包。可以通过增大内核参数 net.core.netdev_budget 和 net.core.netdev_budget_usecs 的值来修复

inbound packets dropped ratio

该值为网卡丢包与接受包之比，该值在 0.1% 以上就会报告警告，在 2% 以上就会严重警告。

根据图表描述可知影响该值的有：

  • softnet 的 backlog 满载
  • 错误的虚拟局域网标签
  • 错误的协议类型
  • IPv6 的数据帧发送到了不支持 IPv6 的服务器上
